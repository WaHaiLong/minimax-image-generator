# Apple 风格界面设计规范 v1.0

## 设计理念

> **一个界面，一个焦点。**
> 用户打开界面 → 知道该做什么 → 做完得到满意结果。
> 不要让用户思考。

---

## 1. 色彩系统

### 调色板
```
背景色：     #FFFFFF（主背景）
次背景：     #F5F5F7（卡片/区块）
分割线：     rgba(0,0,0,0.06)
文字主色：   #1D1D1F（接近纯黑，Apple 用这个）
文字次色：   #86868B（辅助说明）
强调色：     #0071E3（Apple 蓝，用于主按钮/链接）
强调色 hover：#0077ED
成功色：     #34C759
错误色：     #FF3B30
禁用色：     #AEAEB2
```

### 对比现有设计
- 现有：深色 `#07070e` + 紫色 `#8b5cf6` → 刺眼、工具感强
- 新版：白色 + Apple 蓝 → 干净、信任感强

```

```

## 3. 布局结构

### 整体布局（单页应用）
```
┌─────────────────────────────────────────────┐
│  顶部导航栏（56px，白色，固定）                  │
│  [Logo]  图片生成 | 语音合成 | 画廊            │
├─────────────────────────────────────────────┤
│                                             │
│           主内容区（居中，最大 680px）           │
│                                             │
└─────────────────────────────────────────────┘
```

**核心改变**：
- 移除左侧 220px 侧边栏 → 内容区增加了 220px 宽度
- 主内容区居中，最大宽度 680px → 不撑满屏幕，留白感
- 顶部导航固定 → 始终可见，切换流畅

### 图片生成 Tab 布局（上下流）
```
步骤 1：风格参考（上传参考图区域）
        ↓
步骤 2：提示词输入（大大大的文本框）
        ↓
步骤 3：生成（一个巨大的主按钮）
        ↓
步骤 4：预览（生成的图片，占满宽度）
```

不再左侧图+右侧设置 → 改为**线性流程**，用户按顺序往下走。

## 4. 导航栏设计

```html
高度：56px
背景：白色 + 底部 1px 分割线
Logo：MiniMax AI（左边）
Tab：  图片生成 | 语音合成 | 画廊（居中）
```

Tab 切换：
- 当前 Tab：文字加粗（600）+ Apple 蓝 + 底部 2px 蓝线
- 非当前 Tab：文字次色，正常字重
- Hover：文字变深

## 5. 风格参考区（新增功能区）

```
┌─────────────────────────────────────────────┐
│                                             │
│   📷 上传参考图，获得相似风格                   │
│                                             │
│   ┌ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ┐                  │
│     拖拽图片到这里 或 点击上传                    │
│     支持 JPG / PNG / WebP                     │
│   └ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ┘                  │
│                                             │
│   已提取风格：                                 │
│   [电影感 ×] [暖色调 ×] [柔焦 ×]               │
│                                             │
└─────────────────────────────────────────────┘
```

**状态**：
- 空状态：虚线框 + 上传图标 + 说明文字
- 上传中：进度条
- 分析完成：图片缩略图 + 风格标签（可删除）
- 错误：红色提示 + 重试按钮

风格标签：`border-radius: 9999px; padding: 4px 12px; background: #F5F5F7;`

- 旧的"下载"、"复制"作为次级按钮，放在图片下方

## 11. 语音合成 Tab

同样遵循单列流程：
```
1. 文本输入（大文本框）
2. 音色选择（横向卡片，不是 grid）
3. 设置（折叠区域，默认收起）
4. 生成按钮
5. 预览播放器
```

## 13. 组件汇总

### 按钮
| 类型 | 样式 |
|------|------|
| 主按钮 | 蓝底白字，圆角 12px，52px 高 |
| 次按钮 | 白底灰边黑字，hover 灰底 |
| 文字按钮 | 无背景，蓝色文字，hover 下划线 |
| 图标按钮 | 32px 圆角，透明底，hover 灰底 |

### 输入框
- 圆角 12px
- 边框 `#E5E5EA`，focus `#0071E3` + 蓝色光晕
- 内边距 14px 16px

### 卡片
- 圆角 16px
- 背景 `#F5F5F7` 或白色
- 阴影：`0 2px 12px rgba(0,0,0,0.08)`

### 标签
- 圆角 9999px（胶囊形）
- `padding: 4px 12px`
- `background: #F5F5F7; color: #1D1D1F;`

## 14. 实现优先级

**Phase 1（基础重构）**：
1. 新的 variables.css（颜色/字体/间距）
2. 新的 components.css（Apple 风格组件）
3. 新的 index.html 布局框架（顶部 Tab + 单列内容）
4. 导航切换逻辑

**Phase 2（功能迁移）**：
5. 图片生成界面（单列流程）
6. 风格参考上传区
7. 快速风格卡片（替代 preset tags）
8. 历史记录精简版

**Phase 3（其他 Tab）**：
9. 语音合成 Apple 风格
10. 画廊 Apple 风格

**Phase 4（新增功能）**：
11. 风格参考 API 对接

## 15. 不兼容说明

**需要删除的内容**：
- `variables.css` 中的深色变量（`--bg-base`, `--bg-surface` 等）
- `components.css` 中的暗色组件样式（`.layout`, `.header`, `.sidebar`）
- `index.html` 中的左侧 sidebar HTML
- 旧的深色主题所有相关 CSS

**新增文件**：
- `css/apple-variables.css`（Apple 风格变量）
- `css/apple-components.css`（Apple 风格组件）
- 更新 `index.html`（新布局）

**建议**：先在 `css/` 下新建 `apple-*.css`，确认没问题后再替换旧文件。
